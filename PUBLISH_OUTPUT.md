# PUBLISH_OUTPUT

This file describes how to publish the **client-library** generated by `cdd-kotlin` and how to keep it synchronized with its source OpenAPI specification.

## Setting up an Auto-Updating Client Library

You can set up a GitHub Action cron job in the generated client repository to automatically fetch the latest OpenAPI server specification and regenerate the client.

```yaml
name: Auto-Sync Client

on:
  schedule:
    # Run daily at 00:00 UTC
    - cron: '0 0 * * *'
  workflow_dispatch:

jobs:
  sync:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout Client Repo
        uses: actions/checkout@v6

      - name: Setup CDD Kotlin
        run: |
          wget https://github.com/offscale/cdd-kotlin/releases/latest/download/cdd-kotlin.tar
          tar -xf cdd-kotlin.tar

      - name: Fetch Latest OpenAPI Spec
        run: curl -sL https://api.myserver.com/openapi.json -o openapi.json

      - name: Regenerate Models and Routes
        run: ./cdd-kotlin/bin/cdd-kotlin merge_openapi -s openapi.json -d src/main/kotlin

      - name: Detect Changes
        id: git-diff
        run: echo "changes=$(git diff --quiet && echo 0 || echo 1)" >> $GITHUB_ENV

      - name: Commit and Push Updates
        if: env.changes == '1'
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "41898282+github-actions[bot]@users.noreply.github.com"
          git add src/main/kotlin
          git commit -m "chore: auto-sync client with latest OpenAPI spec"
          git push
```

## Publishing the Generated Client

Once generated and pushed, you can publish the client (which is a standard Kotlin Multiplatform or JVM project) to Maven Central or a private repository like GitHub Packages using the standard Gradle publish task.
