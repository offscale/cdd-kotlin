# Goal: Implement Live HTTP Integration Tests for cdd-kotlin

We have recently upgraded our central E2E test harness (`../local-test.sh`). The harness now includes a **Coverage Hook** and a **Chaos Audit**. 

Currently, the test suites generated by cdd-kotlin only perform local instantiations or mocked assertions (e.g., `expect(service).toBeDefined()`). As a result, the new test harness fails the build because it detects 0% HTTP traffic coverage reaching the Docker `petstore_server`.

### Your Task:
Modify the code generator templates inside this directory (`cdd-kotlin`) so that the resulting SDK test suite dispatches **real, live network requests** against all endpoints defined in the OpenAPI spec.

**Requirements for the Generated Tests:**
1. **Target URL:** All requests must target `http://localhost:8080/v2`.
2. **Payloads:** Use dummy payloads that fulfill the schema requirements (strings, ints, etc.) so the endpoints execute successfully.
3. **Assertions:** The generated tests must assert that the HTTP Status Code is valid and that the payload successfully deserializes. (If they don`t, the Chaos Audit in the harness will intentionally fail your build).

### How to Verify Your Work:
Run the overarching E2E harness restricted to your language from the root directory:
```bash
cd ..
ONLY_TEST=cdd-kotlin ./local-test.sh only-test
```

If your generator is correctly outputting live network calls for every endpoint, the `verify_coverage.py` script will report **"Coverage Validation Passed: 100% of endpoints were tested."**
